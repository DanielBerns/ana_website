Here is Draft 1 of the Public Proxy API Specification, designed with strict role separation between the Human Interface and the Machine (Ana) Interface.

### 1. Security & Identity Management
Because this website sits on the public internet and holds system state, authentication must differentiate between the two types of actors:

* **Human Users (Web Frontend):**
    * Authenticate via standard OAuth 2.0 / OpenID Connect (e.g., using Auth0, Keycloak, or a custom identity provider).
    * Receive short-lived JSON Web Tokens (JWTs) in HTTP-only, secure cookies to prevent XSS.
* **Ana System (Machine-to-Machine):**
    * Authenticates using a high-entropy, long-lived API Key (provisioned out-of-band and stored securely in Ana's local environment variables), *or* strictly enforced Mutual TLS (mTLS) using client certificates.
    * Operates under a strict "System Role" that bypasses human UI limits but is restricted to specific sync endpoints.

### 2. Task & Command Mailbox (The Queue)
Human users enqueue work; Ana dequeues it.

**`POST /api/v1/tasks` (Human Only)**
* **Purpose:** A user requests Ana to perform an action (e.g., "Analyze weather station data").
* **Payload:** JSON containing the command type and parameters.
* **Response:** `201 Created`. Returns a `task_id` with status `PENDING`.

**`GET /api/v1/tasks/pending` (Ana Only)**
* **Purpose:** Ana's outbound `TimeNode` (scheduler) triggers an adapter to poll this endpoint to see if human users have requested any work.
* **Response:** `200 OK` with a JSON array of pending tasks.

**`PATCH /api/v1/tasks/{task_id}/status` (Ana Only)**
* **Purpose:** Ana acknowledges receipt of a task (updating it to `IN_PROGRESS`) and later marks it `COMPLETED` or `FAILED`.
* **Payload:** `{"status": "IN_PROGRESS", "correlation_id": "internal-uuid"}`

### 3. Resource Management
Resources are the raw data files or intermediate payloads.

**`POST /api/v1/resources` (Ana Only)**
* **Purpose:** When Ana's internal IONodes fetch or generate new raw data, she uploads it here for persistence or human review.
* **Payload:** Multipart form-data or binary stream.
* **Response:** `201 Created` with a public `resource_uri`.

**`GET /api/v1/resources` (Human & Ana)**
* **Purpose:** List available resources. Humans can browse what Ana has uploaded; Ana can list resources uploaded by humans.
* **Query Params:** `?source=ana` or `?source=human`.

**`DELETE /api/v1/resources/{resource_id}` (Human Only)**
* **Purpose:** Administrative cleanup of old data.
* **Response:** `204 No Content`.

### 4. Knowledge & Reports
Reports are the final, deduced outcomes generated by Ana's Reasoners.

**`POST /api/v1/reports` (Ana Only)**
* **Purpose:** Ana's `ReporterNode` uploads the final generated JSON/PDF system report to the public proxy.
* **Payload:** Report metadata and the file blob.
* **Response:** `201 Created` with `report_id`.

**`GET /api/v1/reports/{report_id}` (Human Only)**
* **Purpose:** Humans download the deduced intelligence.
* **Response:** `200 OK` returning the file.

### 2. Storage Quota Enforcement Strategy

Because of the 350 MB limit, we cannot simply rely on intercepting the `Content-Length` header, as clients can easily spoof it. Instead, Flask needs a middleware or request hook to actively stream and enforce the limit.

Here is the architectural flow for handling uploads safely:

1. **Calculate Available Space:** Before accepting any file bytes, query the database: 
   `total_used = SUM(Resource.file_size_bytes) + SUM(Report.file_size_bytes)`.
   `available_space = 367,000,000 - total_used`.
2. **Stream and Truncate:** When receiving a `POST /api/v1/resources` from Ana, use Flask's `request.stream` to read the file in chunks (e.g., 8KB chunks).
3. **Hard Reject:** If the accumulated chunk bytes exceed `available_space`, immediately close the connection, delete the partially written file from the OS, and return `413 Payload Too Large`.
4. **Commit:** Only if the stream completes within the limit do you write the file to the final OS path and commit the `Resource` row to SQLite.

### 1. Security & Authentication Decorators

```python
from functools import wraps
from flask import request, jsonify, current_app
# Assume a decode_jwt function exists for the human tokens

def require_human_jwt(f):
    """Protects endpoints meant only for manually provisioned human users."""
    @wraps(f)
    def decorated_function(*args, **kwargs):
        auth_header = request.headers.get('Authorization')
        if not auth_header or not auth_header.startswith('Bearer '):
            return jsonify({"error": "Missing or invalid token"}), 401
        
        token = auth_header.split(" ")[1]
        user_payload = decode_jwt(token) # Validates signature and expiry
        if not user_payload:
            return jsonify({"error": "Unauthorized"}), 401
            
        # Inject the user ID into the request context for the route to use
        request.current_user_id = user_payload['user_id']
        return f(*args, **kwargs)
    return decorated_function

def require_ana_system(f):
    """Protects endpoints meant exclusively for Ana's local adapters."""
    @wraps(f)
    def decorated_function(*args, **kwargs):
        auth_header = request.headers.get('Authorization')
        expected_key = current_app.config.get('ANA_SYSTEM_API_KEY')
        
        # Strict exact-match validation
        if not auth_header or auth_header != f"Bearer {expected_key}":
            return jsonify({"error": "Forbidden: System Access Only"}), 403
            
        return f(*args, **kwargs)
    return decorated_function
```

### 2. The 350 MB Storage Enforcer
To prevent malicious or accidental disk exhaustion, we intercept the stream before Flask fully caches large files. 

```python
import os
from sqlalchemy import func

MAX_STORAGE_BYTES = 367_000_000  # ~350 MB

def get_available_space(db_session) -> int:
    """Calculates exactly how much space is left based on the database truth."""
    res_used = db_session.query(func.sum(Resource.file_size_bytes)).scalar() or 0
    rep_used = db_session.query(func.sum(Report.file_size_bytes)).scalar() or 0
    return MAX_STORAGE_BYTES - (res_used + rep_used)

def safe_chunked_upload(flask_request, target_path: str, available_space: int) -> int:
    """
    Reads the incoming multipart/form-data stream in chunks. 
    Aborts and deletes the partial file if the limit is exceeded.
    """
    file_obj = flask_request.files.get('file')
    if not file_obj:
        raise ValueError("No file provided")

    bytes_written = 0
    chunk_size = 8192 # 8 KB chunks

    try:
        with open(target_path, 'wb') as f:
            while True:
                chunk = file_obj.stream.read(chunk_size)
                if not chunk:
                    break
                
                bytes_written += len(chunk)
                if bytes_written > available_space:
                    f.close()
                    os.remove(target_path) # Clean up the partial file
                    raise IOError("Storage quota exceeded during upload.")
                
                f.write(chunk)
        return bytes_written
    except Exception as e:
        if os.path.exists(target_path):
            os.remove(target_path)
        raise e
```

### 2. The Pydantic Flask Middleware
To keep our route definitions clean, we will create a custom decorator that intercepts the request, passes the JSON to the Pydantic model, and returns a detailed `400 Bad Request` if validation fails.

```python
from functools import wraps
from flask import request, jsonify
from pydantic import ValidationError

def validate_json(model_class):
    """
    Decorator to parse incoming JSON against a Pydantic model.
    Injects the validated Pydantic object into the route function.
    """
    def decorator(f):
        @wraps(f)
        def decorated_function(*args, **kwargs):
            if not request.is_json:
                return jsonify({"error": "Request Content-Type must be application/json"}), 415
            
            try:
                # Parse and validate the payload
                validated_data = model_class(**request.get_json())
            except ValidationError as e:
                # Return Pydantic's highly detailed error messages to the client
                return jsonify({
                    "error": "Payload validation failed",
                    "details": e.errors()
                }), 400
            
            # Inject the validated object as a keyword argument
            kwargs['payload'] = validated_data
            return f(*args, **kwargs)
        return decorated_function
    return decorator
```

import json

@api.route('/reports', methods=['POST'])
@require_ana_system
def upload_report():
    """Ana uploads the final deductions and an optional file."""
    
    # 1. Extract and validate the JSON metadata using Pydantic
    raw_metadata = request.form.get('metadata')
    if not raw_metadata:
        return jsonify({"error": "Missing 'metadata' field in form data"}), 400
        
    try:
        parsed_dict = json.loads(raw_metadata)
        metadata: ReportMetadataSchema = ReportMetadataSchema(**parsed_dict)
    except (json.JSONDecodeError, ValidationError) as e:
        return jsonify({"error": "Invalid metadata structure", "details": str(e)}), 400

    # 2. Check for the file and enforce the 350 MB quota constraint
    file_size = 0
    storage_path = None
    
    if 'file' in request.files:
        available_space = get_available_space(db_session)
        target_filename = f"report_{metadata.triggering_task_id}.pdf"
        storage_path = os.path.join("storage/reports", target_filename)
        
        try:
            # Reusing the safety enforcer from Draft 3
            file_size = safe_chunked_upload(request, storage_path, available_space)
        except IOError as e:
            return jsonify({"error": str(e)}), 413

    # 3. Commit to SQLAlchemy
    new_report = Report(
        task_id=metadata.triggering_task_id,
        title=metadata.title,
        deductions=[d.model_dump() for d in metadata.deductions],
        file_size_bytes=file_size,
        storage_path=storage_path
    )

    return jsonify({"report_id": new_report.id, "message": "Report saved"}), 201
